# Narrative Review 006 — Re-centering: cards + glyph bank are the core; diagrams are optional

- **Date:** 2026-06-18
- **Turn / prompt:** The occult diagrams should **not** be the center — they're optional tools players
  may bring in. The **core** is *drawing cards to infuse glass beads with significance* and *applying
  glyphs from the glyph bank*.
- **What was produced:** named the **two core actions** at the top of [GAME_LOOP](../GAME_LOOP.md);
  defined the **glyph bank** as a core always-available resource in [SYMBOL_SETS](../SYMBOL_SETS.md#the-glyph-bank);
  demoted Tree of Life from default skill-tree topology to an **optional overlay** in
  [PROGRESSION](../PROGRESSION.md) (default = plain branch tree; default ranks = Castalian);
  reframed [AESTHETIC](../AESTHETIC.md#occult-diagrams-optional-tools-not-the-core) occult diagrams as
  opt-in; updated mirror-warden + PLAN (Phase 2 core = cards+glyph bank; diagrams optional in Phase 4).

## Analysis (against the study materials)

- **Course-correction was warranted (Shipp ch.2, "where theme begins"):** I had let an *opt-in*
  layer (occult diagrams) creep toward the core — exactly the trap Shipp warns of (don't let the
  theme live in the opt-in layer). The core now sits in the **baked-in** layer: two legible verbs on
  glass beads. Healthier.
- **Verb clarity (Tom Smith):** The game's identity is now two crisp verbs — *infuse (a card)* and
  *apply (a glyph)* — both acting on a bead to give it significance. This is a stronger, more
  teachable core than "use the Tree of Life."
- **Always-a-move (Shipp ch.4):** The glyph bank being **always available** adds a second floor
  beside Pass — you can always apply a glyph. Reinforces the no-deadlock guarantee.
- **Accessibility/onboarding:** A new player meets a glyph palette + a card hand, not a Qabalistic
  diagram. The esoteric trees become a *reward/opt-in* for the curious — better difficulty ramp.

## Risks / watch-items

- **Don't lose the depth:** the optional diagrams are genuinely good content; keep them as first-class
  *options* (well-built, not vestigial), just not gating. (mirror-warden)
- **Glyph bank vs. cards balance:** define what each contributes so neither trivializes the other —
  cards = sourced *meaning/lore* (drawn, limited); glyphs = *correspondence qualifiers* (free, always
  on). Watch that free glyphs don't make card-draw pointless. (crucible-engineer + bead-smith)
- **Glyph gating:** some glyphs unlock by circuit/skill — keep a generous always-available starter set
  so the core action is never starved.

## Suggestions

1. **bead-smith:** specify the **glyph bank** contents + which are starter vs. gated; ensure the
   starter set alone supports full play.
2. **crucible-engineer:** model *infuse-card* and *apply-glyph* as the two primary moves; the
   skill-tree view is presentation-swappable (branch tree default, Tree-of-Life optional) over the
   same node data.
3. **bead-smith:** the correspondence vocabulary can still *borrow values* from CROWLEYDB
   `thelemic_tree.json` without making the Tree itself core.
